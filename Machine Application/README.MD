Machine Application, Тебиев Роберт, 2015

Данное приложение эмулирует взаимодействие покупателя с торговым автоматом. Пользователь может
выбирать действия покупателя. Покупатель может вносить деньги в автомат, выбирать пункты меню
и просить сдачу. Сообщения автомата и результаты действий покупателя отображаются в диалоговых
сообщениях программы.

В начале работы программы вызывается функция LoadConfig, она создаёт экземпляры классов
Machine и Customer, которые соответствуют автомату и покупателю, и возвращает их в качестве
результата через кортеж Pair.

Класс StateBox реализует взаимодействи автомата и покупателя. Пользователю предлагаются диалоговые
сообщения, с помощью которых можно выбирать действия покупателя. В класс Reader вынесены функции
чтения чисел и задержки.

Класс Wallet реализует т.н. кошелёк, который есть и у автомата, и у покупателя. Кошелёк содержит
информацию о количестве монет каждого номинала. Класс автомата не предоставляет доступа к своему
кошелька напрямую, работа с ним возможна через такие методы, как "внести деньги" и "вернуть сдачу".

Класс MenuItem соответствует пункту меню, содержит информацию об имени, цене товара и количестве
единиц данного товара в наличии. Также есть метод, создающий копию экземпляра класса для пользователя,
удаляя из неё информацию о количестве единиц товара в наличии.

Алгоритм поиска монет для выдачи сдачи минимальным числом монет, реализованный в приватном методе
RecGiveChange, использует полный перебор вариантов сдачи. Алгоритм работает для любых возможных
номиналов монет, их количества в автомате и необходимым количеством денег для выдачи сдачи. Класс
Position введён для удобста реализации данного алгоритма.

Для безопасности используются исключения, в большинстве случаев пользователю отображаются сообщения
исключений. При этом метод RecGiveChange и при нахождении решения, и при отсутствии решения завершается
исключением, поскольку метод вызывается рекурсивно и в случае нахождения решения необходимо сразу
выйти из стека рекурсии и передать управление вызывающему методу GiveChange.

В итоге полностью реализован функционал автомата, при этом предусмотрены основные ошибки: у автомата нет
монет на сдачу, у автомата нет выбранного пункта меню, внесённой суммы не хватает для покупки, покупатель
пытается внести монеты, которых у него нет. В случае возникновения этих и других ошибок, пользователь
будет о них оповещён и продолжит работу с приложением.
